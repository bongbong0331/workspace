JSP & Servlet - Session

웹 통신에서 클라이언트가 쿠키를 변조하여 서버에 요청을 보낼 수 있다는 것을 알아봤습니다.
클라이언트가 인증 정보를 변조 할 수 없게 하기 위해 세션(Session) 을 사용합니다.
세션은 인증 정보를 서버에 저장하고 해당 데이터에 접근할 수 있는 키(유추할 수 없는 랜덤한 문자열)를 만들어서 
클라이언트에 전달하는 방식으로 작동합니다.

해당 키는 일반적으로 Session ID 라고 합니다. 브라우저는 해당 키를 쿠키에 저장하고이후에 HTTP 요청을 보낼 때 사용합니다.
서버는 요청에 포함된 키에 해당하는 데이터를 가져와 인증상태를 확인 합니다.


코드 실행 흐름

1. login.jsp 파일 생성하여 사용자에게 ID, PW 와 같은 정보를 받을 form 태그를 생성하자 . 



쿠키에는 여러분의 세션 정보 key 값이(유추 할수없는 랜덤 데이터) 있고
이를 통하여 서버에서는 이용자를 식별하고 인증처리를 합니다. 

이용자의 인증 상태를 훔칠 수 있는데, 이를 세션 하이재킹( Session Hijacking ) 이라고 합니다.
